1. 针对11位移动电话 (mobile)；
2. 针对带3位区号加8位座机号的固定电话（tel）；
3. 针对带4位区号加7或8位座机号的固定电话（tel）；
4. 针对以上2和3场景加1到4位的分机号（tel）；
5. 针对10位的400或800电话(corp)；
6. 针对以上所有场景，允许固话区号和座机号以及分机号之间通过空格或者横杠分开；允许手机号在满足344，335，353，443，434，533（甚至满足前3后3在一起之后中间可以任意拆分）的前提下使用空格或者横杠分开；允许400或800电话在满足 334，343，433的前提下使用空格或者横杠分开。

注意：

    1. 移动电话直接转化成仅11位数字的字符串。
    2. 固话转化成 区号-座机号-分机号 格式。
    3. 400或800转成 xxx-xxx-xxxx 结构。
    4. 因为固定电话长度不固定，固话内部或者手机号内部允许通过横杠-和空格 来分割，所以如果固话在前的话，固话跟其他联系方式之间又通过横杠或者空格分割，会导致识别出错（将固话补了分机号，后面的联系方式因为被识别到分机号了而丢失）
    5. 如果提供的字符串超过已知的长度，会自动截取前n位返回，比如13位的1开头的也会有返回，返回的是前11位。
    6. 不带区号的固话不识别。
